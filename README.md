# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

#### Данные и типы данных (Layer: Data)
В проекте используются следующие интерфейсы и типы для описания сущностей:

#### Сущность приложения
IProduct - описывает карточку товара:
id - уникальный идентификатор;
description - описание товара;
image - ссылка на изображение;
title - название;
category - категория товара;
price - цена;

IBuyer - описывает контактные данные покупателя:
payment - способ оплаты;
email, phone, address - контактная информация.

#### Интерфейсы для работы с API 
`IApi` - базовый интерфейс для сетевых запросов(get, post);
`IProductResponse` - формат ответа сервера со списком товаров (содержит массив items и число total);
`IOrder` - расширяет IBuyer, добавляя список ID купленных товаров и итоговую сумму;
`IOrderResult` - ответ сервера при успешном заказе (содержит id транзакции и total).

#### Класс Catalog
Отвечает за хранение и работу с каталогом товара, полученных с сервера.

Данные: список товаров (list) и текущий выбранный товар currentProduct.
 
`_list` - массив всех товаров;
`_currentProduct` - товар выбранный для просмотра в модальном отклоненный.

Методы класса: 
`save(items)` - сохраняет прошедший массив товаров в модель.
`getAll() `- возвращает полный список товаров.
`getProductById(id)` - ищет товар в списке по его идентификатору.
`saveProduct(product)` - сохраняет товар в модель как "выбранный".

#### Класс Basket
Управляет списком товаров, которые пользователь добавил в корзину.

Данные: массив выбранных товаров (items).
`_items` -  массив товаров, добавленных к покупке.

Методы: 
`add(product)` - добавляет товар в корзину.
`delete(id)` - удаляет товар из корзины по его id.
`clear()` - полностью очищает корзину.
`getPrice()` - рассчитывает суммарную стоимость всех товаров.
`getCountProduct()` - возвращает количество товаров в корзине.
`hasProduct(id)` - проверяет, находится ли уже конкретный товар в корзине.

#### Класс Buyer
Хранит и валидирует контактные данные покупателя и способ оплаты.

Данные: 
`_payment, _email, _phone, _address` - защищенные поля данных.

Методы:
`saveBuyerData(data)` - сохраняет данные. Можно обновить как все поля сразу, так и только одно.
`getBuyerData()` - возвращает объект со всеми текущими данными покупателя.
`validate()` - проверяет поля на пустоту. Возвраает объект с текстом ошибок для каждого невалидного поля.
`isValid()` - возвращает true, если в объекте ошибок пусто.
`clearBuyerData()` - сбрасывает все данные.

#### Слой коммуникации
Класс `WebLarekApi` - обеспечивает взаимодействие прилоожения с бэкэндом магазина.

Конструктор: принимает экземпляр класса API и CDN_URL (для формирования полных путей к изображениям).

Методы: 
`getProducts(): Promise<IProduct[]>` - получает список товаров с сервера.
`orderProducts(order: IOrder): Promise<IOrderResult>` - отправляет данные о заказе и возвращает результат оформления.

##### Слой представления (View)
Все компоненты представления наследутся от базового класса `Component<T>`. Они отвечают за отображение интерфейса, установку  данных в DOM - элементы иигенерацию событий при взаимодействии с пользователем.

Базовые компоненты:
`Components` : абстрактные базовый класс. Содержат общую логику для работы с Dom (установка текста, изображений, переключения классов, управление блокировкой кнопок). 

`Modal:` универсальная модальное окно. Управляет открытием или закрытием и установкой контента точка не имеет наследников. Генерирует события modal:open и modal:close.
Form: базовый класс для форм. Обрабатывает ввод данных, отображает ошибки валидации и управляет активностью кнопки сабмита.

2. Компоненты страниц и блоков 

`Page`: класс для управления главной страницы точка содержит галерею товаров иконку корзины со счётчиком и управляет блокировкой прокрутки всей страницы. 
Basket: класс для отображения корзины. Управляет списком товаров, отображением общей стоимости и составлением кнопки оформить. 
 
3. Группа классов Card карточки

Имеют общего родителя Card, который содержит логику отображения заголовка и цены. 

`Card`: описывает общие поля (заголовок, цена). 
`CardCatalog`: используется для отображения в главной галерее (добавляет категорию и изображение).
`CardPreview`: используется для детального просмотра(добавляет описание, кнопку «в корзину/ удалить»).
`CardBasket`: используется для отображения в списке корзины (добавляет порядковый номер).

4. Группа классов Form
Наследуется от базового класса Form.

`OrderForm`: форма первого шага точка позволяет выбрать способ оплаты и ввести адрес доставки .
ContactsForm: форма второго шага заказа точка предназначена для ввода email и номера телефона.

5. Компонент Success
`Success`: отображает сообщение об успешном оформлении заказа и итоговую сумму списания.

Событийная модель 
Взаимодействие между слоями View и Model происходит через Презентер с помощью событий.

События изменения данных (Model):

`items:changed` - Вызывается при обновлении списка товаров в модели каталога точка презентер перерисовывает главную страницу;
`basket:changed` - Вызывается при любом изменении в корзине (добавление, удаление, очистка). Презендер обновляет счётчик на странице и содержимое корзины;
`formErrors:change` - Вызывается при изменении состояния валидации данных покупателя точка презентер обновляет текст ошибок и активность кнопок форумах.

События взаимодействия с интерфейсом (View):

`card:select` - Крик по карточке в каталоге; 
`product:add` - добавление товаров в корзину; 
`product:delete` - Закрытие модального окна;
`basket:open` - нажатие на иконку корзины; 
`order:open` - переход к оформлению заказа из корзины;
`order:submit` - отправка формы заказа; 
`contacts:submit` - отправка контактных данных и завершение оплаты;
`modal:open` — открытие любого модального окна (используется для блокировки прокрутки страницы);
`modal:close` — закрытие модального окна (разблокировка прокрутки);
`/^(order|contacts)\..*:change/` — универсальное событие ввода данных в поля форм. Обновляет соответствующие поля в модели Buyer.
